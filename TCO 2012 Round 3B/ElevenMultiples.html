TCO 2012 Round 3B 1 500 ElevenMultiples

Problem Statement

John and Brus think 4 and 7 are lucky numbers. They think 11, the sum of 4 and 7, is a very lucky number. They decided to make a multiple of 11.

They went to a shop and bought N pieces of strings of digits. You are given String[] pieces containing N elements. The i-th element of pieces is the string of digits written on the i-th piece they bought. They want to make a multiple of 11 by concatenating all N pieces. The pieces are painted with different colors, so it's possible to distinguish two pieces even if they contain the same strings of digits. There are exactly N! ways to concatenate the pieces. How many of them are divisible by 11? Return this number modulo 1,000,000,007.

Definition

  • Method countMultiples
  • Parameters vector<string>
  • Returns int

Limits

  • Time limit (s) 2.000
  • Memory limit (MB) 64

Constraints

  • pieces will contain between 1 and 50 elements, inclusive.
  • Each element of pieces will contain between 1 and 50 characters, inclusive.
  • Each character in pieces will be a digit ('0'-'9').
  • No element of pieces will start with zero ('0').

Test cases

  1. input
    • pieces { "58", "2012", "123" }
    output
    Returns 2
    note
    There are 6 ways to concatenate the pieces: "582012123", "581232012", "201258123", "201212358", "123582012", and "123201258". Only "582012123" and "201258123" are divisible by 11.
  2. input
    • pieces { "1", "1111", "1", "11" }
    output
    Returns 24
    note
    There are 24 ways to concatenate the pieces. Even though all of them result in the same number "11111111", they are considered distinct. This number is divisible by 11.
